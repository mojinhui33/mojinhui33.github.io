欢迎使用 OPC REST 风格的 Web 服务接口

OPC Web Services 平台是开放的 SOA 架构平台，Web Services 则是 SOA 体系和核心。目前在三种主流的 Web Services 实现方案中，因为 REST 模式的 Web Services 与复杂的 SOAP 和 XML-RPC 对比来讲明显的更加简洁，越来越多的 Web 服务开始采用 REST 风格设计和实现。

通过利用 REST 自身的 HATEOAS(Hypermedia As The Engine Of Application State, 超媒体驱动的应用状态变更) 特性，Web Services 富有体系化的结构和易扩展的特点，让OPC的业务支撑能力更加丰富多彩，同时也使其可扩展性、可订制能力极大增强。通过 REST 风格的 Web Services，几乎能够将系统中的任何功能集成到我们所期盼的软件中，以应对今天和将来各式各样的业务融合所带来的机遇和挑战。

  • Jersey Client
  • Android SDK

鉴于 RESTful Web Services 的访问方式非常简单，本文档仅以 Java 语言为例，介绍访问服务的方法。其他语言/系统访问 REST 服务的方式与此类似，在互联网上可查到大量说明文档，本文不再赘述。
